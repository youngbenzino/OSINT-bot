# OSINT-bot

OSINT-bot — Python-бот для сбора и анализа информации из Telegram-каналов. Позволяет получать сообщения, статистику и метаданные каналов для анализа публичных данных, исследований или образовательных целей.

---

## Содержание репозитория

- `main.py` — основной скрипт для запуска бота  
- `config.py` — конфигурационный файл с токенами и ключами API (НЕ хранить в публичных репозиториях)  
- `.gitignore` — исключает из репозитория приватные файлы, сессии и IDE-конфиги  
- `*.session` — файлы сессий Telegram API (создаются автоматически Telethon)  

> Важно: `config.py` и файлы `*.session` не должны попадать в публичный репозиторий.

---

## Требования

- Python 3.10 или выше  
- Установить зависимости: `pip install telethon` (или `pip install -r requirements.txt`, если есть файл)

---

## Настройка

1. Создайте локально в корне проекта файл `config.py` со следующими значениями (ЗАПОЛНИТЕ своими ключами, не коммите этот файл): `api_id = 'ВАШ_API_ID'`, `api_hash = 'ВАШ_API_HASH'`, `bot_token = 'ВАШ_BOT_TOKEN'`.  
2. Убедитесь, что в `.gitignore` есть строки: `*.session`, `config.py`, `.idea/`. Это не даст случайно запушить сессии и секреты.

---

## Быстрый запуск

1. Активируйте виртуальное окружение (рекомендуется).  
2. Установите зависимости: `pip install telethon`.  
3. Запустите бота: `python main.py`.  

Бот подключится к Telegram и начнёт работать в соответствии с логикой в `main.py` (обрабатывать команды, собирать посты/комментарии, выводить или сохранять результаты).

---

## Пример использования (логика)

- Подключение к API по `api_id`/`api_hash` и запуск с бот-токеном.  
- Получение сообщений из канала: в коде используется `client.iter_messages('channel_username', limit=...)`.  
- Сбор комментариев, сопоставление авторов, генерация ссылок на комментарии и подсчёт статистики.  
(Подробности — в `main.py`; при необходимости можно вынести фичи в отдельные модули: `collector.py`, `analyzer.py`, `storage.py`.)

---

## Безопасность и рекомендации

1. **Не храните** настоящие ключи (`api_id`, `api_hash`, `bot_token`) в публичных репозиториях.  
2. Если секреты случайно попали в репозиторий — **незамедлительно** переместите их локально, выполните `git filter-repo`/BFG для очистки истории и смените скомпрометированные ключи.  
3. Удалите все локальные файлы сессий `*.session` и завершите активные сессии в приложении Telegram: `Settings → Devices → Terminate all other sessions`.  
4. Для хранения секретов используйте переменные окружения или файл `.env` (сам `.env` заносите в `.gitignore`), либо секретный менеджер CI/CD.

---

## Что именно нельзя оставлять в репозитории

- `config.py` с реальными ключами  
- любые `*.session` (telethon session files)  
- файлы типа `secrets.json`, `credentials.yml` и т.п., где содержатся токены или пароли

- Сохранять результаты в CSV/JSON или БД (Postgres/SQLite) для последующей аналитики.

## Лицензия и правовой аспект

Проект предназначен для образовательных и исследовательских целей. Использование данных чужих приватных каналов без разрешения и автоматизированный сбор чужих персональных данных может нарушать законы и правила Telegram. Пользоваться на свой страх и риск.
